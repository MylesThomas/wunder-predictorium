## 2. Set up your environment and packages

We strongly recommend using a virtual environment to keep your project dependencies tidy and avoid conflicts. Probably any Python 3.10+ will do. Submissions will be evaluated in a Docker container based on `python:3.11-slim-bookworm` image.

### Option A: Using uv (Recommended - Fast!)

[uv](https://github.com/astral-sh/uv) is a fast Python package installer and resolver.

```bash
# Install uv if you haven't already
curl -LsSf https://astral.sh/uv/install.sh | sh

# Install dependencies (automatically creates virtual environment)
uv sync

# Activate the environment
source .venv/bin/activate  # macOS/Linux
# or
.venv\Scripts\activate  # Windows
```

### Option B: Traditional venv + pip

**macOS/Linux**
```bash
python -m venv env
source env/bin/activate
```

**Windows**
```bash
python -m venv env
env\Scripts\activate
```

### Then install the required packages:

```bash
pip install numpy pandas pyarrow tqdm onnxruntime torch
```

## 3. Explore the data

The dataset is provided in Parquet format.

- `datasets/train.parquet` â€” training data (10,721 sequences)
- `datasets/valid.parquet` â€” validation data (1,444 sequences)

Each sequence has **1000 steps**. The first 99 steps are for warm-up/context, and you predict for steps 99-999.

## 4. Run the example solution

We provided a baseline solution in `example_solution` folder.

```bash
cd example_solution
python solution.py
```

This script will load the validation data and the pre-trained `baseline.onnx` model, run inference, and calculate the Pearson Correlation score.

> **FUN FACT**: This baseline was generated by an LLM Assistant. We asked it to create a Vanilla GRU model and optimized for inference speed (ONNX) to meet the competition's time limits. It was trained on the full training dataset and tuned using the provided validation set.

## 5. Develop your own model

1. Create your own `solution.py` implementing the `PredictionModel` class.
2. Train your model using `datasets/train.parquet`.
3. Validate using `datasets/valid.parquet`.
4. Export your trained model (e.g., to ONNX) for fast inference.

Good luck with the challenge! We're excited to see what you build.

## 6. Submit your solution

### Quick submission

```bash
# Package your solution
zip -r submission.zip solution.py your_model.onnx

# Go to submit page
open https://predictorium.wundernn.io/submit
```
